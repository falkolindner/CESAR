# Contributing

Thanks for contributing.

## Development workflow

1. Create a feature branch from `main`.
2. Keep changes focused and small.
3. Run checks locally before opening a PR.

## Required checks

- `npm run lint`
- `npm test`
- `npm run prettier:verify`

## Commit guidance

- Use clear commit messages that explain intent.
- Include tests for logic changes when possible.
- Avoid committing secrets, certificates, or org-local cache files.

## Pull requests

- Describe what changed and why.
- Include screenshots when UI behavior changes.
- Mention any Salesforce org prerequisites or metadata dependencies.
